## 07-1. 클래스의 정의와 인스턴스의 생성

### 클래스 = 데이터(Data) + 메소드(Method)
- **클래스(Class)**: 데이터(멤버 변수)와 그 데이터를 다루는 메소드를 하나로 묶은 설계도.
- **인스턴스(Instance)**: 클래스라는 설계도로 실제로 메모리에 찍어낸 실체.
  ```java
  BankAccount yoon = new BankAccount();  // new 로 인스턴스 생성
  ```
- **C 비교** 🔧: C의 `struct`는 데이터만 묶지만, 자바 클래스는 **데이터 + 그 데이터를 다루는 함수(메소드)까지** 한 덩어리로 묶는다. 이게 OOP의 출발점.

### 참조변수(Reference Variable)의 특성
- 인스턴스는 `new`로 **힙(heap)** 에 생성되고, 참조변수는 그 인스턴스를 **가리키는(참조하는)** 값을 담는다.
- **C 비교** 🔧: 참조변수는 C의 **포인터**와 비슷하지만, 주소를 직접 계산·조작할 수 없고 오직 "가리키기"만 한다.
- 참조변수를 **메소드의 매개변수**로 선언하면, 인스턴스의 참조 값을 넘겨 그 인스턴스를 함수 안에서 다룰 수 있다.
  ```java
  public static void check(BankAccount acc) {  // 참조를 매개변수로
      acc.checkMyBalance();
  }
  ```

### 참조변수에 null 대입
- 참조변수가 아무 인스턴스도 가리키지 않게 하려면 `null`을 대입한다.
  ```java
  ref = null;              // 참조하던 인스턴스와의 관계를 끊음
  if (ref == null) { ... } // 참조하는 인스턴스가 없는지 확인
  ```

### 📎 참고: 생성되는 클래스 파일의 수
- 확장자가 `.class`인 파일은 **정의된 클래스의 수만큼** 생성된다.
- 예) `PassingRef.java` 안에 `BankAccount`, `PassingRef` 두 클래스가 있으면 → `BankAccount.class`, `PassingRef.class` 두 개 생성.

## 07-2. 생성자(Constructor)와 String 클래스의 소개

### 생성자(Constructor)
인스턴스를 생성하면서 **멤버 변수를 원하는 값으로 초기화**하는 특별한 메소드.

```java
public BankAccount(String acc, String ss, int bal) {  // 생성자
    accNumber = acc;   // 멤버 변수 초기화
    ssNumber  = ss;
    balance   = bal;
}
// 호출: new BankAccount("12-34-89", "990990-9090990", 10000);
```

**⭐ 생성자의 두 가지 규칙 (반드시 기억)**
- **생성자의 이름은 클래스의 이름과 동일해야 한다.**
- **생성자는 값을 반환하지 않고 반환형도 표시하지 않는다.** (`void`조차 안 씀!)

**⭐ 인스턴스 생성과 생성자 (핵심 명제)**
> "인스턴스 생성의 마지막 단계는 생성자 호출이다."
>
> "어떠한 이유로든 생성자 호출이 생략된 인스턴스는 인스턴스가 아니다."

즉 `new`로 인스턴스를 만들 때 **반드시 생성자가 호출**되며, 이 과정을 거치지 않은 것은 인스턴스로 인정되지 않는다.

### 디폴트 생성자(Default Constructor)
- 생성자를 **하나도 정의하지 않으면**, 자바 컴파일러가 **디폴트 생성자**를 자동으로 넣어준다.
  ```java
  public BankAccount() {   // 컴파일러가 자동 삽입하는 '디폴트 생성자'
      // empty
  }
  ```
- 특징:
  - **인자를 전달받지 않는** 형태로 삽입된다.
  - 내부적으로 **하는 일이 없다**(비어 있음).
  - 덕분에 생성자를 안 만들어도 "생성자 호출"이라는 인스턴스 생성 규칙은 유지된다.
- ⚠️ **하지만 디폴트 생성자에 의존하지 말고, 생성자는 직접 정의해 주는 것이 좋다.**
  아주 예외적인 상황이 아니라면, **생성자가 필요 없는 클래스는 잘 정의된 클래스가 아닐 확률이 높기 때문**이다. (초기화가 필요 없는 데이터 묶음은 드물다.)

### String 클래스에 대한 첫 소개
- 문자열은 큰따옴표로 표현하고, `String` 참조변수로 참조할 수 있다.
  ```java
  String myName = "Yoon Sung Woo";  // String형 참조변수의 문자열 참조
  ```
- `String`은 사실 **자바가 제공하는 클래스의 이름**. 지금은 "문자열을 참조하는 참조변수" 정도로 이해하면 충분(자세한 건 이후 장에서).

## 07-3. 자바의 이름 규칙(Naming Rule)

코드 가독성을 위해 통용되는 이름 짓기 관례.

| 대상 | 규칙 | 예시 |
|------|------|------|
| **클래스** | Camel Case — **첫 문자 대문자**, 이어지는 단어도 대문자 시작 | `Circle` + `Point` = `CirclePoint` |
| **메소드·변수** | 변형된 Camel Case — **첫 문자 소문자**, 이어지는 단어는 대문자 | `add`+`Your`+`Money` = `addYourMoney` |
| **상수** | **모든 문자 대문자**, 단어 사이는 언더바(`_`)로 연결 | `final int COLOR_RAINBOW = 7;` |

- **C 비교** 🔧: C에서 상수를 `#define MAX_SIZE 100`처럼 대문자로 쓰던 관례와 동일. 자바는 `final`로 상수를 만들고 이름은 대문자로.

## ✅ 이 장의 정리

1. **클래스 = 데이터 + 메소드**. `new`로 힙에 **인스턴스**를 만들고 **참조변수**로 가리킨다. (C의 struct+포인터의 진화형)
2. **생성자**는 ① 이름이 클래스명과 같고 ② 반환형이 없다. 인스턴스 생성의 **마지막 단계가 생성자 호출**이다.
3. 생성자를 안 만들면 컴파일러가 **디폴트 생성자**(빈 무인자 생성자)를 넣어주지만, **직접 정의하는 습관**을 들이자.
4. 이름 규칙: 클래스=`PascalStyle`, 메소드/변수=`camelStyle`, 상수=`ALL_CAPS`.
