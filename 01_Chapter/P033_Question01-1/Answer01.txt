[문제 01-1] 클래스의 이름과 문자열의 출력 - 문제 1

[문제]
앞서 보인 예제 FirstJavaProgram.java를 컴파일하면 FirstJavaProgram.class가
생성된다. 그런데 컴파일 시 생성되는 클래스 파일의 이름을 변경하고 싶다.
예를 들어 SimpleJavaProgram.class가 생성되게 하려면 무엇을 수정해야 되겠는가?
나름대로 예측해 보자.

[답안]
소스 코드 안의 "클래스 이름"을 SimpleJavaProgram으로 수정한다.

    class FirstJavaProgram      →      class SimpleJavaProgram

[이유]
.class 파일의 이름은 .java 파일의 이름이 아니라, 소스 코드 안에 선언된
클래스의 이름에서 결정된다. 컴파일러(javac)는 파일명이 아니라 'class OOO'
선언을 보고 OOO.class 를 생성하기 때문이다.

따라서 파일 이름만 mv 등으로 바꾸는 것으로는 목적을 달성할 수 없다.
(예: SimpleJavaProgram.java 안에 class FirstJavaProgram 이 들어있으면,
 컴파일 결과는 여전히 FirstJavaProgram.class 가 생성된다.)

[보충]
- 위 예제의 클래스는 public 이 아니므로(그냥 class), 파일 이름은 클래스
  이름과 달라도 컴파일은 된다. 다만 일치시키는 것이 표준 관례다.
- 만약 public class 였다면, 파일 이름도 반드시 클래스 이름과 똑같이
  (SimpleJavaProgram.java) 맞춰야 컴파일 에러가 나지 않는다.
- 정리: 클래스 이름 수정은 '필수', 파일 이름 수정은 '권장(public이면 필수)'.

[공식 답안]
클래스의 이름을 SimpleJavaProgram으로 수정한다.
