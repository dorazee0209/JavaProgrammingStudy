# 문제 09-1 — 필드 초기화식 vs 생성자 호출 순서

`Circle`이 `Point`를 필드로 활용하도록 고치는 과정에서 발견한 버그와 그 원인 정리.

## 시도 1 — 생성자 안에서 `this.x`를 미리 씀

```java
public Circle(int x, int y, int rad) {
    Point center = new Point(this.x, this.y);   // this.x, this.y 사용
    this.rad = rad;
    this.x = x;
    this.y = y;
}
```

결과: `[0, 0]`.

- **원인 1 — 대입 전 참조**: `this.x`는 매개변수 `x`가 아니라 **필드** `x`를 가리키는데,
  이 줄이 실행되는 시점엔 아직 `this.x = x;` 대입이 일어나기 전이라 필드는 기본값 `0`
  그대로다. 즉 `new Point(0, 0)`을 만들어버린 것.
- **원인 2 — 지역 변수로 선언됨**: 순서만 고쳐도(`this.x = x;`를 먼저 실행) 근본적으로
  안 되는 이유가 하나 더 있다. `Point center = new Point(...)`가 **생성자 몸통 안에서
  선언된 지역 변수**라는 점이다. 클래스 위쪽에 `Point center;` 필드 선언이 없으니, 이
  `center`는 생성자가 끝나는 순간 사라진다. `showCircleInfo()` 같은 다른 메소드에서는
  애초에 `center`라는 이름 자체를 알 수 없다(컴파일 에러) — **필드 선언 자체가 빠져
  있는 게 더 근본적인 원인**이다.

## 시도 2 — 필드 초기화식으로 옮김

```java
class Circle {
    int rad;
    int x, y;

    public Circle(int x, int y, int rad) {
        this.rad = rad;
        this.x = x;
        this.y = y;
    }
    Point pnt = new Point(this.x, this.y);   // 필드 초기화식으로 이동
    ...
}
```

결과: 여전히 `[0, 0]`.

소스코드 상으로는 `pnt` 필드가 생성자 아래에 있어서 나중에 실행될 것처럼 보이지만,
**필드 초기화식은 텍스트 위치와 무관하게 항상 생성자 몸통(body)보다 먼저 실행**된다.

## 원인 — 인스턴스 생성 순서

> "인스턴스 생성의 마지막 단계는 생성자 호출이다." ([07_Chapter README.md](../../07_Chapter/README.md) 참고)

실제 인스턴스 생성 순서:

| 순서 | 단계 | 이 예제에서 |
|---|---|---|
| 1 | 필드 기본값 세팅 | `x=0, y=0, rad=0, pnt=null` |
| 2 | 필드 초기화식 실행 (선언 순서대로) | `pnt = new Point(this.x, this.y)` → 이 시점 x,y는 아직 0 |
| 3 | **생성자 호출 (진짜 마지막 단계)** | `this.x = x; this.y = y;` 등 실행 |

필드 초기화식은 "인스턴스 생성 과정"의 일부라서 2단계에서 이미 끝나버리고, 생성자
몸통 코드는 그보다 뒤인 3단계에서 실행되기 때문에 순서가 어긋난다.

C에는 이런 "필드 초기화식" 개념 자체가 없어서(구조체 멤버는 전부 명시적으로 대입) 이런
순서 문제가 발생할 여지가 없다 — Java 특유의 함정.

### 필드 기본값 vs 로컬 변수

- `int` 등 **인스턴스 필드**는 값을 안 주면 1단계에서 자동으로 기본값(`0`, 참조형은
  `null`, boolean은 `false`)이 들어간다.
- 반면 **로컬 변수**(메소드 안에서 선언한 변수)는 기본값이 없다 — 초기화 안 하고 쓰면
  `variable might not have been initialized` 컴파일 에러가 난다. 자동 기본값은 필드에만
  적용되는 규칙.

## 개선 방향

- `Point`를 필드로 두려면 선언(`Point center;`)과 **값 대입은 분리**해야 한다.
- 값 대입(`new Point(...)`)은 반드시 **생성자 몸통 안**에서, 매개변수 `x, y`를 직접
  사용해 처리한다 (`this.`를 거치지 않아도 매개변수는 호출 시점 값을 이미 갖고 있음).
- 필드 초기화식에는 매개변수에 의존하는 초기값을 넣지 않는다 — 매개변수가 없는
  시점(클래스 로드/필드 세팅 시점)에 실행되므로.
