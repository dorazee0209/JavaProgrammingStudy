# 열혈 Java 프로그래밍 — 훑어보기 학습 가이드

> 윤성우 저 「열혈 Java 프로그래밍」 (전 34장, 908p). Java SE 기본기를 매우 친절하게 풀어쓴 입문서로, 변수·OOP·컬렉션·제네릭·람다/스트림·I/O·멀티스레드까지 "순수 자바"의 토대를 한 권으로 깔아준다. Spring·웹·DB는 다루지 않는다(아래 5번 참고).

이 가이드는 **2027 상반기 백엔드(Java/Spring) 신입 취업**을 목표로 하는 CS 편입생(4-1 종료 직후, 직접 손으로 짠 프로젝트 사실상 0) 관점에서, "만들었다"가 아니라 "한 줄 한 줄 설명할 수 있다"를 기준으로 재정렬했다.

## 백엔드 취준 우선순위 그룹핑

같은 "필수"라도 백엔드 면접/실무 빈도가 다르다. 신입 관점에서 4묶음으로 재정렬했다.

### A. 반드시 탄탄히 (필수 — 코드로 설명 가능해야 함)
> OOP 핵심 + 컬렉션 + 예외 + 메모리/동등성 + 람다·스트림 + 멀티스레드. 면접 단골이자 Spring 이해의 직접 토대.

- **7, 9, 10, 11** — 클래스/인스턴스, 캡슐화, static, 오버로딩+String. OOP와 엔티티·DTO·유틸 설계의 출발점. 특히 11장 `==` vs `equals`, String 불변성은 면접 빈출.
- **14, 15, 16, 17** — 상속·오버라이딩·다형성·인터페이스/추상클래스. "구현이 아닌 약속(타입)으로 프로그래밍"이 Spring DI·전략 설계의 뿌리.
- **18** — 예외처리. try-with-resources, Checked/Unchecked 구분은 트랜잭션·자원관리의 기본.
- **19** — JVM 메모리 모델 + Object(equals/hashCode/toString). GC·동등성은 면접 단골, HashMap/HashSet과 직결.
- **21** — 제네릭 1. 컬렉션을 제대로 쓰려면 필수.
- **23** — 컬렉션 프레임워크 1. **가장 중요한 단원.** List/Set/Map 특성, hashCode/equals, Comparable/Comparator는 거의 매 면접 등장.
- **27, 29, 30** — 람다 + 스트림 1·2. 실무 데이터 가공에서 매일 쓰는 `filter/map/collect/groupingBy`.
- **32** — I/O 스트림. 파일·네트워크 I/O와 데코레이터 패턴의 근간.
- **34** — 쓰레드·동기화. 동시성은 백엔드 서버의 핵심이자 면접 단골(race condition, 쓰레드 풀, synchronized).

### B. 중요 (1회독 후 복습 — 깊게는 나중에)
- **3, 4** — 상수/형변환, 연산자. 기초지만 비트/지수표기 등은 가볍게.
- **8** — 패키지/클래스패스. Spring 컴포넌트 스캔 이해의 기반. 콘솔 컴파일 실습은 한 번만.
- **12** — 콘솔 입출력(Scanner). 코테 입력 처리에 필요한 정도.
- **20** — 래퍼/BigDecimal/Arrays. BigDecimal(금융 정산)만 확실히.
- **22** — 제네릭 2(와일드카드 PECS). 개념 이해 위주, 실무 빈도는 21장보다 낮음.
- **24** — Collections 알고리즘 + 와일드카드 심화. Comparator 주입 패턴만 확실히.
- **25** — enum/varargs/어노테이션. enum이 클래스라는 점은 실무 빈출.
- **26** — 네스티드 클래스 + 람다 소개. 익명클래스→람다 흐름만 잡으면 27장이 쉬워짐.
- **28** — 메소드 참조 + Optional. Optional은 반환 타입 설계 관점으로 이해.
- **31** — java.time. 엔티티 생성시각·만료 처리에 매일 쓰지만 난이도 낮아 빠르게.

### C. 알아두기 (선택 — 개념만, 깊게 안 파도 됨)
- **33** — NIO/NIO.2. Files 유틸 메서드 정도만. 채널/버퍼는 나중에.

### D. 나중에 / 입사 후 깊게
- **2의 2의 보수·IEEE754 비트 계산**, **4의 비트·쉬프트 연산**, **19의 finalize/clone 심화**, **22·24의 와일드카드 증명 수준**, **34의 ReentrantLock·동기화 컬렉션 세부** — 원리만 알고 넘어가고, 실제로 막혔을 때 다시 파면 된다.

> 1~6장(기초 문법)은 "필수"지만 백엔드 변별력은 낮다. C를 할 줄 알면 **빠르게 통과**하고 7장부터 시간을 쏟는 게 효율적이다.

### 0단계 — 지금(6월 하순): 맛보기 1~5장

## 이 책으로 채워지는 것 vs 안 채워지는 것

**채워지는 것**: 순수 **Java SE 언어 자체**의 기본기 — 문법, OOP(클래스·상속·다형성·인터페이스), 예외, 컬렉션, 제네릭, 람다/스트림, I/O, 멀티스레드. 면접에서 "자바 언어"를 묻는 질문(`==` vs equals, equals/hashCode 규약, String 불변, 오버로딩 vs 오버라이딩, Checked/Unchecked, 동시성)은 이 책으로 대부분 커버된다.

**안 채워지는 것**: 이 책에는 **Spring/Spring Boot, 웹(HTTP·REST·MVC), DB·SQL, JPA/ORM, 빌드툴(Gradle/Maven), 네트워크·서버 실무, 테스트(JUnit), Git 협업 워크플로**가 없다. 즉 이 책만으로는 "백엔드 개발자"가 되지 않는다.

**다음 단계 권장 순서**: 이 책으로 Java를 탄탄히 한 뒤 → **SQL 기초(SELECT/JOIN/인덱스)** → **Spring Boot 입문**(REST API + JPA로 간단한 CRUD 게시판/할일 앱을 직접 손코딩) → **JUnit 테스트** → 그 사이 **코딩테스트**(이 책의 컬렉션·스트림·배열이 그대로 무기). 핵심은 "바이브코딩 말고 직접 짠, 한 줄씩 설명 가능한 프로젝트 1~2개"를 만드는 것이다. 이 책의 23·29·30·34장이 Spring 실무로 직결된다.

## 학습법 팁 (윤성우 책 특성에 맞춤)

1. **소스코드·연습문제 답안 폴더를 적극 활용**하되, 예제는 **눈으로 읽지 말고 직접 타이핑(복붙 금지)**. 윤성우 책은 예제 실행 결과로 개념을 설명하므로, 돌려보고 결과를 예측·검증하는 게 학습의 절반이다.
2. **각 장마다 "왜?"를 한 문장으로 적어라.** 예: "왜 String은 불변인가?", "왜 equals 오버라이딩 시 hashCode도 해야 하나?", "왜 new Thread 대신 쓰레드 풀인가?". 이 "왜"가 곧 면접 답변이다.
3. **watchOut(함정) 항목을 코드로 재현**하라. fall-through, SCE 부수효과 미실행, double `==` 오차, `==` vs equals, `? extends`/`? super` 방향, 지연연산 미실행, race condition 등은 직접 깨뜨려 봐야 기억에 남는다.
4. **손으로 흐름 추적.** 반복문·재귀·업캐스팅/동적바인딩은 종이에 변수 값과 호출 스택을 그려가며 따라가라.
5. **A 그룹 장(7,9,10,11,14-19,21,23,27,29,30,32,34)은 "남에게 설명"** 해보라. 설명이 막히면 그 부분이 아직 안 된 것. "만들었다"가 아니라 "설명할 수 있다"가 기준이다.
6. **B/C 그룹은 1회독 후 넘어가고 막힐 때 복귀.** 완벽주의로 22·24·33장에 시간을 쏟다 컬렉션·스트림·동시성을 못 보면 손해다.
