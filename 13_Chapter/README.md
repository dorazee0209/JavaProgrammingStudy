# Chapter 13 — 배열(Array)

## 배열도 인스턴스다 ⭐

배열은 **`new`로 생성되는 인스턴스(객체)**다. 그냥 값들을 모아 둔 게 아니라, 클래스로부터 인스턴스를 만들 때와 똑같은 방식으로 메모리에 생성된다.

- `new int[5]` → **인스턴스 생성** (다섯 개의 int형 변수로 구성된 인스턴스)
- 인스턴스이기 때문에 **`length`라는 인스턴스 변수**를 가지고 있다. (배열의 길이 정보)
- 참조변수(`ref`)는 이 인스턴스를 **참조**할 뿐, 참조변수 자체가 배열은 아니다.

이 "배열 = 인스턴스"라는 사실이 아래의 참조변수 생성 / 인스턴스 생성이 왜 별개로 일어나는지를 이해하는 핵심이다.

## 배열의 참조변수가 뜻하는 것

```
int   [ ]
 ↑     ↑
int형   배열
변수로
이뤄진
```

`int[]` → **"int형 변수로 이뤄진 배열을 참조하는"** 이라는 의미다.
즉 `int[] ref`는 "int형 변수들로 이뤄진 배열을 참조하는 `ref`" 라고 읽으면 된다.

## 배열의 생성 — 참조변수 vs 인스턴스

```java
int[] ref = new int[5];
```

이 한 줄은 사실 **두 가지 일**을 동시에 한다.

| 코드 조각 | 하는 일 |
|---|---|
| `int[] ref` | **참조변수** `ref` 생성 |
| `new int[5]` | **배열 인스턴스** 생성 (다섯 개의 int형 변수로 구성) |

```
int[] ref = new int[5] ;
     │ 참조변수 생성      │ 인스턴스 생성
     ↓                    ↓
   [ref]  ---참조--->  [□□□□□]  ← length = 5 (인스턴스 변수)
```

앞서 본 것처럼 **참조변수 생성**과 **인스턴스 생성**은 별개의 일이고, 인스턴스 쪽에 `length`가 붙어 있다.

## 배열을 생성과 동시에 초기화하기

기본 생성:
```java
int[] arr = new int[3];
```

초기화할 값을 중괄호로 나열하고 싶을 때, **가장 헷갈리는 함정**이 여기 있다.

```java
int[] arr = new int[3] {1, 2, 3};   // ❌ 컴파일 오류!
```

⚠️ 초기화 값의 개수(3개)로 배열의 길이를 이미 계산할 수 있기 때문에, **길이 `[3]`을 생략해야 한다**는 규칙이 있다. 대괄호 안에 숫자와 중괄호 초기화를 동시에 쓰면 오류가 난다.

```java
int[] arr = new int[] {1, 2, 3};    // ✅ 올바른 형태, length는 3
```

그리고 이 문장은 다음과 같이 더 줄여 쓸 수 있다.

```java
int[] arr = {1, 2, 3};              // ✅ 위와 완전히 동일 (length = 3)
```

### 정리 — 초기화 3형제

| 코드 | 가능 여부 | 비고 |
|---|---|---|
| `new int[3] {1, 2, 3}` | ❌ | 길이와 중괄호 초기화 동시 사용 불가 |
| `new int[] {1, 2, 3}` | ✅ | 길이 생략, 정식 형태 |
| `{1, 2, 3}` | ✅ | `new int[]` 생략한 축약형 |

## 참조변수 선언의 두 가지 방법

대괄호 `[]`는 타입 뒤(`int[] arr`)와 변수명 뒤(`int arr[]`) 둘 다 올 수 있고, **완전히 동일한 의미**다.

```java
int[] arr;      // 방법 1
int arr[];      // 방법 2 (동일)
```

배열 생성 문장에서도 마찬가지다.

```java
int[] ar = new int[3];   // 조금 더 선호하는 방법 ✅
int ar[] = new int[3];   // 문법적으로 동일하지만 덜 선호됨
```

📌 **결론**: 문법적으로는 둘 다 맞지만, **`int[] arr` 형태(타입 뒤에 `[]`)가 더 선호되는 스타일**이다. "int형 배열"이라는 타입을 한 덩어리로 보여주기 때문.

## main의 `String[] args` — 왜 0번째가 프로그램 이름이 아닐까?

`main(String[] args)`의 `args`도 배열(`String[]`)이다. 여기엔 **실행 시 넘긴 인자만** 담긴다.

```bash
java P295_Simple arg1 arg2
# → args[0] = "arg1", args[1] = "arg2"   (프로그램 이름 없음!)
```

시스템 프로그래밍을 해봤다면 이게 어색할 수 있다. 유닉스에선 프로세스의 인자 벡터(`argv`) 0번째에 **실행 파일 이름 자체**가 들어가는 게 전통이기 때문이다. 같은 걸 C로 짜서 `gcc`로 빌드해 돌려보면 차이가 선명하다.

```c
int main(int argc, char *argv[]) { ... }   // argv = argument vector
```
```bash
gcc demo.c -o a.out
./a.out arg1 arg2
# → argv[0] = "./a.out"   ← 실행 파일 이름이 0번째!
#   argv[1] = "arg1"
#   argv[2] = "arg2"
```

C에선 **OS가 띄운 실행 파일 = 내 프로그램**이라, 그 이름(`./a.out`)이 곧장 `argv[0]`에 담긴다.

### 핵심: 자바는 실행 주체가 한 겹 더 있다

OS가 실제로 띄우는 프로세스는 우리 클래스가 아니라 **`java` 런처(JVM)** 다. OS 레벨에서 보면:

```
argv[0] = "java"          ← OS가 띄운 실행 파일
argv[1] = "P295_Simple"   ← 실행할 클래스 이름
argv[2] = "arg1"
argv[3] = "arg2"
```

**JVM이 이걸 먼저 파싱**해서 "클래스 이름(`P295_Simple`)"을 골라 쓰고, **그 뒤에 남은 것만** `main`의 `args`로 넘겨준다. 그래서 우리가 받는 `args`는 OS의 `argv`가 아니라 **JVM이 한 번 걸러낸 결과물**이다.

### 왜 굳이 이렇게?

인자 벡터에 프로그램 이름을 넣는 건 프로그램이 **자기 자신을 알 수단**을 주기 위해서다. 그런데 자바는 그 정보를 리플렉션으로 이미 알 수 있다.

```java
System.out.println(P295_Simple.class.getName());  // 클래스 이름은 따로 얻으면 됨
```

그러니 `args`엔 이름을 끼워 넣을 이유가 없다 → **순수하게 "사용자가 넘긴 인자"만** 담자는 설계.

| | 인자 벡터 0번째 | 이유 |
|---|---|---|
| C (유닉스 `argv`) | 프로그램 이름 포함 | OS가 그대로 전달, 자기 인식 수단이 이것뿐 |
| Java (`args`) | 첫 번째 실제 인자부터 | `java` 런처가 클래스명을 걸러냄 · 클래스명은 리플렉션으로 별도 취득 |

## 다차원 배열

배열의 배열. 대표적으로 **2차원 배열**은 "행(row) × 열(column)"의 표 형태 데이터를 담는다.

### 2차원 배열의 생성

```java
int[][] arr = new int[3][4];   // 3행 4열, 총 12개의 int 공간
```

- `arr.length` → **행(row)의 개수** (3)
- `arr[0].length` → **열(column)의 개수** (4)
- 각 요소 접근은 `arr[행][열]` — 예: `arr[1][2]`

핵심은 2차원 배열도 결국 **"1차원 배열들을 참조하는 1차원 배열"**이라는 점이다. `arr`는 길이 3짜리 배열이고, 그 각 칸(`arr[0]`, `arr[1]`, `arr[2]`)이 다시 길이 4짜리 배열을 참조한다.

### 2차원 배열의 초기화

1차원 배열과 마찬가지로, 2차원 배열도 **생성과 동시에 초기화**할 수 있다.

```java
int[][] arr = new int[][] {
    {11, 22, 33},   // 1행 초기화
    {44, 55, 66},   // 2행 초기화
    {77, 88, 99}    // 3행 초기화
};
```

이 문장은 가로·세로 길이가 모두 3인 2차원 배열을 만들면서 모든 요소를 지정 값으로 채운다. 그리고 1차원 때처럼 `new int[][]`를 생략한 **축약형**도 가능하다.

```java
int[][] arr = {
    {11, 22, 33},
    {44, 55, 66},
    {77, 88, 99}
};   // 위와 완전히 동일
```

### 가변 배열 — 행마다 길이가 달라도 된다

그렇다면 각 행의 중괄호에 담긴 값의 개수를 다르게 하면 어떻게 될까?

```java
int[][] arr = {
    {11},
    {22, 33},
    {44, 55, 66}
};
```

행마다 길이가 제각각인 **가변 배열(ragged array)**이 만들어진다. "2차원 배열 = 1차원 배열들을 참조하는 배열"이기 때문에, 참조되는 각 배열의 길이가 서로 달라도 전혀 문제없다.

| | 참조하는 1차원 배열 | 길이 (`arr[i].length`) |
|---|---|---|
| `arr[0]` | `{11}` | 1 |
| `arr[1]` | `{22, 33}` | 2 |
| `arr[2]` | `{44, 55, 66}` | 3 |

> 실측: `arr.length`는 3(행 개수), 각 행의 `length`는 1·2·3으로 서로 다르게 나온다. ✅

### 정리 📌

- 배열은 `new`로 생성되는 **인스턴스** — 그래서 `length`라는 인스턴스 변수를 가진다.
- `int[] ref = new int[5];` 한 줄 = **참조변수 생성 + 인스턴스 생성**, 두 가지 일.
- 생성과 동시에 초기화는 `new int[] {1, 2, 3}` 또는 축약형 `{1, 2, 3}` — 길이와 중괄호 동시 사용(`new int[3] {1,2,3}`)은 ❌.
- 선언은 `int[] arr` 스타일 선호 — "int형 배열"이라는 타입이 한 덩어리로 보이기 때문.
- `main`의 `args`엔 프로그램 이름이 없다 — OS가 띄운 건 `java` 런처이고, 런처가 클래스명을 걸러낸 뒤 **순수 사용자 인자만** 넘겨준다.
- 2차원 배열은 **"1차원 배열들을 참조하는 1차원 배열"** — `arr.length`=행 수, `arr[i].length`=그 행의 열 수. 그래서 행마다 길이가 다른 **가변 배열**도 가능하다.
